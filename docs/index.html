GLOBE Cloud Insights

An interactive dashboard and reproducible data pipeline for the NASA GLOBE Observer Clouds 2022 Challenge, drawing on more than forty-two thousand cloud observations contributed by volunteers across eighty-nine countries.

Section IOverview

A complete, citable record of citizen-science cloud observations made readable, mappable, and downloadable, with every quality filter and provenance step documented in code.

Between 15 January and 15 February 2022, volunteers around the world recorded what they saw above them through the NASA GLOBE Observer Cloud Challenge. They identified cloud genera, estimated cloud cover, and submitted timestamped, geolocated observations from every continent. This repository takes the resulting record and turns it into an interactive dashboard, a reproducible Python pipeline, and a small open dataset that anyone can study, teach with, or extend.

A note on the bundled sample

To keep the repository light and to ensure the dashboard runs immediately for any first-time visitor, a 500-row sample of the campaign record is committed to data/raw/. The figures in Section III labelled Sample are computed from that file. The pipeline in Section IV downloads the full record on demand, with checksum-cached weekly chunks.

Section IIDashboard Views

Four panels, each tuned to one question a reader might bring to the data.

World Map

Every observation is plotted on an interactive globe. Points cluster automatically into hotspots so that a single overcast morning in Bavaria does not crowd out a thinner record from Patagonia. Hovering surfaces the timestamp, the reported sky condition, and the dominant cloud genus.

Timeline

Daily counts of submissions across the thirty-two-day campaign. Peaks correspond to school days and to organized events; valleys correspond to weekends and to cloud-free deserts. The timeline reads like a heartbeat of the volunteer network.

Patterns & Distributions

Three small multiples sit beside each other: a donut of sky conditions, a histogram of cloud cover percentages, and a bar of the most active contributing countries. The donut uses a fixed warm palette so that the same condition keeps the same colour across every view.

Browse & Download

A searchable, sortable table of raw observations, with one-click export to CSV. The download preserves the full schema documented in data/README.md, including the derived cloud-cover percentage and the rule used to infer it from sky-condition labels.

Section IVPipeline & Data

Four stages, each implemented in a single module, each with its own test file. The pipeline is reproducible from the GLOBE API to the figure on the page.

Stage 1

Fetch

GLOBE API in weekly chunks, with SHA-256 checksum caching. Module: fetch.py.

Stage 2

Clean

Coordinate validation, timestamp parsing, derived cloud-cover percentage. Module: clean.py.

Stage 3

Persist

Tidy, analysis-ready columns written to Parquet for fast notebook reads. Output: data/processed/.

Stage 4

Analyse

Descriptive and inferential statistics, mapped via Plotly and Folium. Module: analysis.py.

Stage 5

Explore

Streamlit dashboard with filters, maps, charts, export. Entry point: app/streamlit_app.py.

Quality filters

  • Rows missing latitude or longitude are dropped.
  • Coordinates outside −90 to 90 latitude or −180 to 180 longitude are dropped.
  • Rows without a valid timestamp are dropped.
  • Cloud cover percentage is inferred from sky condition where missing — Clear 0%, Few 15%, Scattered 40%, Broken 70%, Overcast 95%, Obscured 100%.

Ethics note on location privacy

Coordinates are publicly available under the GLOBE Program’s open-data policy. Even so, sensitive inference about individual observers — particularly at high spatial resolution — is discouraged. The dashboard clusters points to the regional scale by default for this reason.
